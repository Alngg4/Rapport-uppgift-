# Rapport-uppgift-
Gemensam slutsats

I denna uppgift har jag arbetat med tre olika skriptspråk – Bash, PowerShell och Python – för att visa hur säkerhetskontroller kan genomföras och automatiseras i olika miljöer. Varje skript är anpassat efter sitt respektive operativsystem och användningsområde, men tillsammans bildar de en helhetslösning för grundläggande IT-säkerhetsarbete.

Bash-skriptet är riktat mot Linux-miljöer och fokuserar på systemnära kontroller, såsom analys av autentiseringsloggar, granskning av aktiv nätverkstrafik samt kontroll av systemets uppdateringsstatus. Detta skript visar hur enkla kommandon kan användas för att snabbt få en överblick över systemets säkerhetsläge och samtidigt skapa spårbar dokumentation i form av en säkerhetsrapport.

PowerShell-skriptet är anpassat för Windows-miljöer och använder inbyggda funktioner för att analysera händelser i Event Viewer, kontrollera att brandväggen är aktiverad samt verifiera att säkerhetsuppdateringar är installerade. Genom detta skript visar jag hur administrativa säkerhetskontroller kan automatiseras på ett strukturerat och lättförståeligt sätt i Windows.

Python-skriptet fungerar som en plattformsoberoende analysdel och knyter ihop flera olika datakällor, såsom processlistor, loggfiler och nätverksdata. Genom att använda en modulär struktur med tydliga funktioner kan analysen enkelt anpassas och vidareutvecklas. Det simulerade API-anropet visar även hur extern säkerhetsinformation kan integreras för att komplettera den lokala analysen.

Sammantaget visar dessa tre skript hur olika tekniker och verktyg kan användas tillsammans för att skapa ett mer heltäckande och systematiskt säkerhetsarbete. Lösningen är medvetet utformad för att vara enkel att förstå, men samtidigt tillräckligt strukturerad för att spegla verkliga arbetssätt inom IT-säkerhet. Genom att kombinera flera miljöer och språk demonstrerar uppgiften både teknisk bredd och förståelse för hur automatisering kan bidra till ökad säkerhet och bättre överblick.
